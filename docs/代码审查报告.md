## 🔴 严重 Bug（会导致运行时错误）

### Bug #1：For Each 循环变量与数组变量冲突（行 ~270）

**位置**：`GenerateQuestions` 中混合运算答案计算部分

```vba
' ❌ 错误代码
Dim nums() As String
nums = Split(parts(0), " + ")
For Each nums In Split(parts(0), " + ")   ' nums 既是数组又是循环变量！
    answerText = Val(answerText) + Val(nums)
Next
```

**问题**：`nums` 被声明为 `String()` 数组，但 `For Each` 的循环变量必须是 `Variant`。这会导致 **类型不匹配运行时错误**。

**修复**：使用独立的 `Variant` 变量作为循环变量，删除冗余代码。

---

### Bug #2：答案计算代码重复执行（行 ~265-280）

**位置**：`GenerateQuestions` 中混合运算答案计算

```vba
' ❌ 错误代码 — 先算一遍
answerText = 0
For Each nums In Split(parts(0), " + ")
    answerText = Val(answerText) + Val(nums)
Next
' 又重新算一遍
answerText = 0
Dim m As Integer
nums = Split(parts(0), " + ")
For m = 0 To UBound(nums)
    answerText = Val(answerText) + Val(nums(m))
Next m
```

**问题**：答案被计算了两次，第一次结果被覆盖。不仅浪费资源，而且第一次计算因为 Bug #1 会报错。

**修复**：删除冗余的第一次计算，只保留 `For` 循环版本。

### Bug #3：GenerateMixed 可能返回空字符串（行 ~165）

**位置**：`GenerateMixed` 函数

```vba
Case 0
    GenerateMixed = GenerateAddition(maxNum, noCarry)
    If GenerateMixed = "" Then GenerateMixed = GenerateAddition(maxNum, False)
```

**问题**：如果 `GenerateAddition` 返回空字符串（不进位模式无法满足时），重试一次 `GenerateAddition(maxNum, False)` 理论上不会返回空，但如果随机数种子问题导致死循环或极端情况，`GenerateMixed` 可能最终为空。后续 `Split(parts(0), ...)` 会因空字符串而出错。

**修复**：在 `GenerateQuestions` 中添加空字符串防护。

## 🟡 一般 Bug（逻辑错误）

### Bug #4：负数判断逻辑无意义（行 ~295）

**位置**：`GenerateQuestions` 中负数处理

```vba
If Rnd() < negativeProb And answerText < 0 Then
    ' 允许负数
ElseIf answerText < 0 Then
    ' 重新生成直到非负
    i = i - 1
    If i < 0 Then i = 0
    GoTo NextIteration
End If
```

**问题**：
1. 对于加法 `a + b`（a,b ≥ 1），结果永远 ≥ 2，不可能为负
2. 对于减法 `a - b`（b ≤ a），结果永远 ≥ 0，不可能为负
3. 对于连减 `a - b - c`（已确保 a-b-c ≥ 0），结果永远 ≥ 0
4. 因此 `answerText < 0` 永远为 False，整个负数判断逻辑永远不会触发

**修复**：移除无意义的负数判断逻辑，简化代码。

---

### Bug #5：`i = i - 1` 后 `GoTo NextIteration` 跳过 `Next i`（行 ~298）

**位置**：`GenerateQuestions` 中

```vba
i = i - 1
If i < 0 Then i = 0
GoTo NextIteration   ' 跳到 NextIteration 标签，然后执行 Next i
```

**问题**：`GoTo NextIteration` 跳到标签后执行 `Next i`，此时 `i` 会先加 1 再判断。如果 `i` 被设为 0，`Next i` 后 `i = 1`，相当于重新生成第 1 题。但如果 `i` 原本是 5，减 1 后为 4，`Next i` 后 `i = 5`，又回到第 5 题——这实际上是正确的重试逻辑。但代码可读性差，且 `If i < 0 Then i = 0` 在正常流程中不会触发（因为 i 从 1 开始，最小减到 0）。

**修复**：保留逻辑但改进注释说明意图。

### Bug #6：GenerateAddition 的 noCarry 检查不完整（行 ~100）

**位置**：`GenerateAddition` 函数

```vba
If noCarry Then
    If (a Mod 10) + (b Mod 10) >= 10 Then
        GenerateAddition = ""
        Exit Function
    End If
End If
```

**问题**：只检查了个位是否进位，但没有检查十位是否进位到百位。例如 `maxNum = 100` 时，`55 + 55 = 110` 超过 100，但个位 `5+5=10` 被检测到。然而如果 `maxNum = 50`，`25 + 25 = 50`，个位 `5+5=10` 被检测，但 `24 + 24 = 48` 个位 `4+4=8` 不检测，这是正确的。问题在于：当 `noCarry = True` 且 `maxNum` 较大时，个位不进位的条件可能过于严格，导致大量重试。

**修复**：保持现有逻辑（对幼升小场景足够），但添加注释说明。

### Bug #7：GenerateChainAdd 的 c 可能等于 0（行 ~140）

**位置**：`GenerateChainAdd` 函数

```vba
c = maxNum - a - b
If c < 1 Then c = 1
```

**问题**：`c` 被强制设为 1 后，`result = a + b + c` 可能超过 `maxNum`。虽然 `Loop While result > maxNum` 会重试，但如果 `a` 和 `b` 都接近 `maxNum/3`，`c` 被设为 1 后结果可能仍然合理。这个逻辑实际上不会导致错误，但 `c` 的值分布不均匀（1 的出现概率偏高）。

**修复**：保持现有逻辑（对幼升小场景可接受）。

---

### Bug #8：全局变量未初始化时可能为 Nothing（行 ~10-12）

**位置**：模块级变量声明

```vba
Dim wsQuestion As Worksheet
Dim wsAnswer As Worksheet
Dim wsRecord As Worksheet
```

**问题**：这些模块级变量在 `Workbook_Open` 中通过 `InitializeSheets` 初始化。但如果用户直接运行 `GenerateQuestions` 而 `Workbook_Open` 未执行（如宏被禁用后手动启用），这些变量可能为 `Nothing`。

**修复**：在 `GenerateQuestions` 开头调用 `InitializeSheets`（已有），但 `InitializeSheets` 中的 `On Error Resume Next` 可能掩盖错误。

**修复**：改进错误处理，确保工作表正确初始化。

## 🟢 建议优化

### Opt #1：参数面板缺少页数设置

**建议**：在 G7/H7 添加"生成页数"参数（默认 1，范围 1-5），支持多页生成。

### Opt #2：布局计算不够精确

**建议**：A4 纵向标准布局应为 5 列 × 10 行 = 50 题/页，当前 `CalculateLayout` 的列数计算不够精确。

### Opt #3：缺少页间分隔

**建议**：多页生成时，页与页之间应有分隔线和分页符。
